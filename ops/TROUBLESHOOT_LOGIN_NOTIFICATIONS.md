# 🔧 Диагностика: Почему не приходят уведомления при Telegram Login

## 📋 Быстрая проверка

### 1. Проверьте логи backend после авторизации

После авторизации в логах backend должны появиться сообщения вида:
```
[AUTH] Successfully sent login notification to user 123456789
```

Или, если есть ошибка:
```
[AUTH] Telegram API error for user 123456789: code=403, description=Forbidden: bot was blocked by the user
[AUTH] User 123456789 hasn't started conversation with bot or blocked the bot
```

**Где смотреть логи:**
- Render: Dashboard → Service → Logs
- Локально: терминал, где запущен backend

### 2. Проверьте настройки домена в BotFather

**Это критично для автоматических уведомлений Telegram Login Widget!**

1. Откройте [@BotFather](https://t.me/BotFather)
2. Отправьте `/setdomain`
3. Выберите вашего бота
4. Проверьте, что домен указан: `deliverty.vercel.app` (без https://, без trailing slash)

**Важно:** Домен должен **точно совпадать** с доменом, где размещен виджет.

### 3. Проверьте, что пользователь начал диалог с ботом

**Проблема:** Telegram Bot API не может отправлять сообщения пользователям, которые не начали диалог с ботом.

**Решение:** Пользователь должен:
1. Найти вашего бота в Telegram (например, `@deliverty_bot`)
2. Нажать кнопку "START" или отправить `/start`
3. После этого бот сможет отправлять уведомления

**Как проверить:**
- Откройте бота в Telegram
- Если видите сообщение от бота (например, приветственное), значит диалог начат ✅
- Если бот пустой, значит нужно начать диалог

## 🔍 Детальная диагностика

### Автоматические уведомления Telegram Login Widget

Telegram Login Widget **автоматически** отправляет уведомления только если:

✅ Домен настроен через `/setdomain` в BotFather  
✅ Домен точно совпадает с доменом сайта  
✅ Используется redirect mode (`data-auth-url`)  
✅ Виджет размещен на правильном домене  

❌ **Не работает если:**
- Домен не настроен или не совпадает
- Используется callback mode (`data-onauth`)
- Виджет на localhost (без ngrok)
- Пользователь авторизуется слишком часто (защита от спама)

### Ручные уведомления (через Bot API)

Backend **автоматически** пытается отправить уведомление через Bot API после успешной авторизации.

**Работает только если:**
✅ Пользователь начал диалог с ботом (`/start`)  
✅ Пользователь не заблокировал бота  
✅ `TG_BOT_TOKEN` настроен правильно  

**Не работает если:**
❌ Пользователь не начал диалог (ошибка 403)  
❌ Пользователь заблокировал бота (ошибка 403)  
❌ Неверный токен бота (ошибка 401)  

## 🐛 Частые ошибки и решения

### Ошибка 403: Forbidden

**Логи:**
```
[AUTH] Telegram API error for user 123456789: code=403, description=Forbidden: bot was blocked by the user
[AUTH] User 123456789 hasn't started conversation with bot or blocked the bot
```

**Причины:**
1. Пользователь не начал диалог с ботом
2. Пользователь заблокировал бота

**Решение:**
- Пользователь должен найти бота в Telegram и нажать "START"
- Или отправить команду `/start` боту

### Ошибка 401: Unauthorized

**Логи:**
```
[AUTH] Telegram API error for user 123456789: code=401, description=Unauthorized
```

**Причины:**
- Неверный `TG_BOT_TOKEN`
- Токен не установлен в переменных окружения

**Решение:**
- Проверьте `TG_BOT_TOKEN` в настройках Render/backend
- Убедитесь, что токен правильный (получить можно через `/token` в @BotFather)

### Нет логов вообще

**Логи:**
```
[AUTH] Cannot send notification: TG_BOT_TOKEN not set
```

**Причины:**
- `TG_BOT_TOKEN` не установлен

**Решение:**
- Установите `TG_BOT_TOKEN` в переменных окружения backend

### Автоматические уведомления не приходят

**Проблема:** Telegram Login Widget не отправляет автоматические уведомления

**Причины:**
1. Домен не настроен в BotFather
2. Домен не совпадает с реальным доменом
3. Используется неправильный режим виджета

**Решение:**
1. Проверьте настройки домена в BotFather (`/setdomain`)
2. Убедитесь, что домен точно совпадает (регистр важен!)
3. Проверьте, что виджет использует `data-auth-url`, а не `data-onauth`


## 🔄 Что делать дальше

1. **Проверьте логи backend** после авторизации - там будет видно, что именно не работает
2. **Если ошибка 403:** Пользователь должен начать диалог с ботом
3. **Если нет ошибок, но уведомления не приходят:** Проверьте настройки домена в BotFather
4. **Если ничего не помогает:** Создайте issue с логами backend

## 💡 Важно понимать

- **Автоматические уведомления** от Telegram Login Widget работают только при правильной настройке домена
- **Ручные уведомления** через Bot API работают только если пользователь начал диалог с ботом
- Оба механизма могут работать одновременно
- Логи backend покажут, какой механизм работает и почему
